# Диагностика проблем по графику потерь (loss)

## 📊 Введение
Когда вы обучаете нейросеть, вы обычно видите только числа потерь (loss). График потерь — как пульс пациента: по его форме можно понять, здоров ли алгоритм и что именно нужно "лечить". Этот файл научит вас быстро распознавать типичные проблемы по форме графика и подскажет конкретные правки в коде.

## Как получить график (быстрая вставка в код)
```python
import matplotlib.pyplot as plt

# Собираем потери в список
losses = []

for i in range(5000):
    # ... ваш код обучения ...
    # loss = ...
    losses.append(loss.item())  # Сохраняем потерю

# Рисуем график
plt.plot(losses)
plt.title('График потерь')
plt.xlabel('Итерация')
plt.ylabel('Loss')
plt.grid(True)

```

### 1) ИДЕАЛЬНЫЙ ГРАФИК ✅

```
Потеря
  ^
  |\
  | \
  |  \_
  |    \_
  |      \_____
  +---------------> Итерации
```

Как выглядит: быстрое падение в начале, затем плавный затухающий спуск, в конце плато.

Характерные признаки:
- Плавный устойчивый спад
- Минимальный шум
- Достижение низкой loss и стабилизация

Причины (в порядке вероятности):
- 60% — Хорошо настроенный LR/оптимизатор
- 25% — Корректная архитектура и инициализация
- 15% — Хороший датасет и нормализация

Решения: ничего менять не нужно — фиксируйте эксперимент и запишите hyperparams.

---

### 2) ГРАФИК РАСТЁТ

```
Потеря
  ^
  |      /
  |     /
  |    /
  |   /
  |  /
  | /
  +---------------> Итерации
```

Как выглядит: loss возрастает с итерациями.

Характерные признаки:
- Чёткий восходящий тренд
- Нет признаков спуска

Причины (в порядке вероятности):
- 90% — Слишком большой LR
- 50% — Ошибка в вычислении loss / обратном проходе
- 30% — Неправильная нормализация входов/меток

Решения (по порядку):
1) Уменьшить LR в 10–100×.
```python
# Было:
optimizer = torch.optim.RMSprop(model.parameters(), lr=0.1)

# Стало:
optimizer = torch.optim.RMSprop(model.parameters(), lr=0.001)
```
2) Проверить правильность `loss.backward()` и `optimizer.step()` и отсутствие двойного применения градиентов.
3) Нормализовать входы/метки (mean/std) и проверить, что лоссы не NaN.

---

### 3) ГРАФИК ПРЫГАЕТ (шум, зубчатый) 

```
Потеря
  ^
  | /\  /\  /\
  |/  \/  \/  \
  |            \_
  +---------------> Итерации
```

Как выглядит: сильные колебания вверх/вниз, нет плавного спада.

Характерные признаки:
- Частые резкие скачки
- Невнятный общий тренд

Причины (в порядке вероятности):
- 70% — Слишком большой LR
- 60% — Очень маленький batch size (шум градиента)
- 30% — Неподходящий оптимизатор / отсутствие момента

Решения:
1) Уменьшить LR в 2–5×.
2) Увеличить batch size.
3) Попробовать `Adam` или добавить momentum.
```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
```
4) Добавить градиентный клиппинг, если градиенты взрываются.

---

### 4) ГРАФИК МЕДЛЕННО ПАДАЕТ 

```
Потеря
  ^
  |\
  | \
  |  \
  |   \________
  +---------------> Итерации
```

Как выглядит: плавный, но очень медленный спад.

Характерные признаки:
- Кривая пологая
- Трудно достичь низкой loss за разумное время

Причины (в порядке вероятности):
- 75% — Слишком маленький LR
- 40% — Оптимизатор требует подстройки
- 30% — Ограниченная ёмкость модели

Решения:
1) Увеличить LR в 3–10× и экспериментировать со scheduler.
2) Попробовать более агрессивный оптимизатор (`Adam` с lr=0.001→0.01).
3) Увеличить ёмкость модели, если после LR улучшения всё ещё медленно.

### 5) БЫСТРО ПАДАЕТ, ПОТОМ ЗАСТРЕВАЕТ (плато)

```
Потеря
  ^
  |\
  | \___
  |     \___
  +---------------> Итерации
```

Как выглядит: ранний быстрый спад, затем долгий застой на высоком уровне.

Характерные признаки:
- Резкий initial drop
- Дальнейшее отсутствие улучшения

Причины (в порядке вероятности):
- 50% — Локальный минимум / малая ёмкость модели
- 40% — Плохо выбранная инициализация
- 30% — Неподходящий оптимизатор/параметры

Решения:
1) Увеличить число нейронов или добавить слой(и).
2) Попробовать инициализации (Xavier, He) и регуляризацию.
3) Использовать learning rate scheduler или рестарт оптимизатора.

### 6) ГРАФИК НЕ ПАДАЕТ ВООБЩЕ (прямая линия)

```
Потеря
  ^
  |________________
  +---------------> Итерации
```

Как выглядит: flat line, loss не меняется.

Характерные признаки:
- Отсутствие обновлений loss
- Градиенты равны нулю или NaN

Причины (в порядке вероятности):
- 50% — Нулевой или слишком маленький LR
- 40% — Замороженные веса / забыли вызвать `optimizer.step()`
- 30% — Мёртвые нейроны (ReLU) или vanishing gradients

Решения:
1) Проверить, что `optimizer.zero_grad()`, `loss.backward()`, `optimizer.step()` вызываются.
2) Убедиться, что LR > 0; попробовать увеличить LR.
3) Сменить активацию (LeakyReLU) и проверить инициализацию.
```python
x = F.leaky_relu(self.layer1(x), 0.01)
```

### 7) ПАДАЕТ, НО ВЫХОДЫ НЕВЕРНЫЕ

```
Потеря
  ^
  |\
  | \
  |  \
  |   \______
  +---------------> Итерации
```

Как выглядит: loss снижается, но метрики качества (accuracy, MAE и т.д.) остаются плохими.

Характерные признаки:
- Низкий loss, но плохие предсказания
- Несоответствие диапазона выходов и меток

Причины (в порядке вероятности):
- 70% — Неправильная активация на выходе
- 40% — Неподходящая функция потерь
- 20% — Ошибка в подготовке/нормализации меток

Решения:
1) Выбрать корректную активацию и loss (например, `sigmoid` + `BCEWithLogitsLoss` для бинарной классификации).
2) Для многоклассовой классификации использовать `CrossEntropyLoss` без предварительного softmax.
3) Для регрессии — убрать активацию на выходе и использовать MSE/MAE.

### 8) ЗАБИТЫЙ ГРАФИК: ДИАГНОСТИКА И ЛЕЧЕНИЕ

```
Потеря
  ^
  |  /\  /\  /\  /\
  | /  \/  \/  \/  \
  |/\  /\  /\  /\  /\
  |  \/  \/  \/  \/  \
  |  /\  /\  /\  /\  /\
  | /  \/  \/  \/  \/  \
  +------------------------> Итерации
```

Характерные признаки:
- Нет плавного движения вниз
- Постоянные хаотичные скачки (иногда от 0 до 1)
- Невозможно увидеть общий тренд — линия похожа на "забор" или шум

Причины (в порядке вероятности):
- 90% — Слишком большой LR (основная причина)
- 70% — Случайный порядок/обучение на одном примере (в выборке большой разброс)
- 50% — Плохая инициализация весов
- 30% — Неправильная активация (ReLU убивает нейроны)

Решения (по порядку):

#### ШАГ 1: УМЕНЬШИТЬ LR В 100 РАЗ
```python
# Было:
optimizer = torch.optim.RMSprop(params=model.parameters(), lr=0.01)

# Стало:
optimizer = torch.optim.RMSprop(params=model.parameters(), lr=0.0001)
```

#### ШАГ 2: СМЕНИТЬ ОПТИМИЗАТОР НА ADAM
```python
optimizer = torch.optim.Adam(params=model.parameters(), lr=0.001)
```

#### ШАГ 3: ИСПОЛЬЗОВАТЬ ВСЕ ПРИМЕРЫ ЗА ЭПОХУ (НЕ RANDOM)
```python
for epoch in range(1000):
    for i in range(length_x):  # Все примеры, не random!
        ret = model(x_train[i])
        loss = error_f(ret, y_train[i].unsqueeze(0))
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

#### ШАГ 4: ДОБАВИТЬ КЛИППИНГ ГРАДИЕНТОВ
```python
loss.backward()
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()
```

#### ШАГ 5: СГЛАДИТЬ ГРАФИК (ТОЛЬКО ДЛЯ ВИЗУАЛИЗАЦИИ)
```python
def smooth(data, window=50):
    smoothed = []
    for i in range(len(data)):
        start = max(0, i - window)
        smoothed.append(sum(data[start:i+1]) / (i - start + 1))
    return smoothed

plt.plot(smooth(losses, 50), color='red', linewidth=2, label='Сглаженный')
plt.plot(losses, color='blue', alpha=0.3, label='Оригинал')
plt.legend()
plt.show()
```

## Быстрая таблица решений (сводка)

Что видите на графике — Что менять (порядок действий)
- Идёт вверх — Снизить LR, проверить loss/backward
- Прыгает — Уменьшить LR, увеличить batch, попробовать Adam
- Медленно падает — Увеличить LR, попробовать scheduler
- Быстро упал и застрял — Увеличить ёмкость модели, сменить инициализацию
- Прямая линия — Проверить backward/step, сменить активацию
- Падает, но ответы неверные — Исправить активацию выходного слоя и loss
- Забитый график — Снижать LR, Adam, весь датасет за эпоху, клиппинг

## Что такое Learning Rate (LR)?
LR — это размер шага при обновлении весов. Слишком большой LR приводит к раскачиванию или росту loss; слишком маленький — очень медленному обучению. Adam с lr=0.001 — хорошая отправная точка.

## Практический алгоритм действий
1. Посмотрите на график и определите его форму.
2. Меняйте по одному параметру: сначала LR, затем архитектуру, затем оптимизатор.
3. Запускайте эксперимент воспроизводимо и фиксируйте результаты (лог с hyperparams).
4. Если не уверены — проверьте, что модель умеет переобучиться на 1–2 примерах (sanity check).

## Главные правила для новичков
- Начинайте с малого: сначала обучите на одном примере, затем на нескольких.
- Меняйте только одну вещь за эксперимент.
- Используйте `Adam` как оптимизатор по умолчанию и lr≈0.001.
- Ведите журнал экспериментов (what/when/result).

## Пример из реальной жизни

Решение: заменить ReLU/без активации на `sigmoid` для бинарной задачи — всё заработало.
